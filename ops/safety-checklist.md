# WCRC 안전 체크리스트

미션 코드 변경을 업로드하거나 현장에서 주행하기 전에 이 목록을 확인한다.

## 업로드 전

- [ ] `git status --short --branch`를 확인했다.
- [ ] 변경 파일과 위험도를 `ops/events.jsonl`에 기록했다.
- [ ] `git diff --check`가 통과했다.
- [ ] Arduino IDE Verify가 통과했다.
- [ ] 전체 자율주행 전에 EEPROM 자세 1,2,3,4,5,7~12가 존재하는지 확인했다.
- [ ] Pixy signature 학습값이 `MissionConfig.h`와 맞는다.
- [ ] Pixy block area가 `missionInstructionMinBlockArea` / `storageMinBlockArea` 기준보다 충분히 크다. 정상 블록이 숨겨지면 기준을 `0`으로 낮춰 먼저 원인을 확인한다.
- [ ] finish 복귀 시간은 현장 테스트 기록 없이 바꾸지 않았다.

## 하드웨어 안전 테스트 순서

- [ ] 바퀴를 띄운 상태에서 주행 방향 테스트.
- [ ] 블록 없이 매니퓰레이터 자세 테스트.
- [ ] 그리퍼 열림/닫힘 테스트.
- [ ] 미션 지시존 미탐지 시 raw Pixy debug를 출력하고 적재함/집기를 건너뛰는지 확인했다.
- [ ] 적재함 미탐지 시 같은 블록을 반복하지 않고 다음 source slot으로 넘어가는지 확인했다.
- [ ] 단일 블록 집기/놓기 테스트.
- [ ] 전체 미션 테스트.

## 안정 후보 기준

- [ ] 전체 현장 주행을 1회 완료했다.
- [ ] 실패 상황에서 기본값으로 계속 진행하지 않고 안전하게 멈춘다.
- [ ] 미션 수행존 배치가 의도한 칸 안에 들어간다.
- [ ] `ops/events.jsonl`만 봐도 롤백할 커밋이나 브랜치가 분명하다.
